在專案根目錄下，開啟終端機並執行：

2.  **模型 ID 偵測**：如果連線成功，會從 LM Studio 返回的 JSON 中解析出目前載入的模型 ID（例如 `qwen2.5-coder-7b-instruct`）。

## 二、將地端模型遷移到內網 PC 的程式修改

本次主要任務是將 AI 推理引擎從雲端 OpenAI 遷移到本地（現已是內網 PC）部署的 LM Studio，並解決過程中遇到的所有兼容性問題。

### 1. 配置變更

*   **檔案**：`src/main/resources/application.properties`
*   **修改內容**：
    *   `chatgpt.apiKey`: 從 `xKcgr_...` 修改為 `ollama` (用於本地端驗證，遠端 API 服務通常不需實際的 Key)。
    *   `chatgpt.apiHost`: 從 `https://api.openai-proxy.com/` 修改為 `http://192.168.137.1:1234/`，指向內網的 LM Studio 伺服器。

### 2. Java 程式碼邏輯修改

#### (1) `Synthesizer.java` (in `csp-service` project)

*   **目的**：調整模型生成的 SQL 語法，使其更符合本地 SQLite 環境。
*   **修改內容**：
    *   在 `instructPrompt` 方法中，將生成的 SQL 類型從「MySQL SQL」改為「SQLite SQL」。
*   **備註**：此修改是透過 `sed` 命令執行，位於 `../csp-service/src/main/java/com/dhf/hrsys/service/impl/Synthesizer.java`。

#### (2) `SseServiceImpl.java` (in `csp-service` project)

*   **目的**：解決 Token 計算 `NullPointerException` 和模型 ID 不匹配問題。
*   **修改內容**：
    *   將 `response.setQuestionTokens(completion.tokens());` 修改為 `response.setQuestionTokens(0);`，避免在非 OpenAI 標準模型下因 Token 計算方式不同而導致的應用程式崩潰。
    *   將 `ChatCompletion.Model.GPT_3_5_TURBO.getName()` 硬編碼的模型名稱 `qwen2.5-coder:7b` 更新為 LM Studio 實際回報的 `qwen2.5-coder-7b-instruct`，確保模型名稱匹配。
*   **備註**：此修改亦透過 `sed` 命令執行，位於 `../csp-service/src/main/java/com/dhf/hrsys/service/impl/SseServiceImpl.java`。

#### (3) `OpenAISSEEventSourceListener.java` (in `csp-service` project)

*   **目的**：解決 Jackson 反序列化 LM Studio 回傳 JSON 時遇到的 `UnrecognizedPropertyException`。
*   **修改內容**：
    *   在 `onEvent` 方法中，在初始化 `ObjectMapper` 後，增加配置 `mapper.configure(com.fasterxml.jackson.databind.DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false);`，使其忽略未知屬性。
*   **備註**：此修改亦透過 `sed` 命令執行，位於 `../csp-service/src/main/java/com/dhf/system/chat/listener/OpenAISSEEventSourceListener.java`。

#### (4) `SqlController.java` (in `csp-portal-web` project)

*   **目的**：解決 LM Studio 生成的 SQL 包含 Markdown 代碼塊標記導致的 `SQLITE_ERROR`。
*   **修改內容**：
    *   在 `processForMyBatis` 方法中，增加了移除 ```sql` 和 ```` 標記的邏輯，確保傳遞給資料庫的是純淨的 SQL 語句。
*   **備註**：此修改位於 `src/main/java/com/dhf/hrsys/controller/SqlController.java`。

## 三、環境配置與偵錯重點

*   **macOS 區域網路權限**：在 macOS 的「隱私權與安全性」->「區域網路」中，務必允許 IntelliJ IDEA 存取區域網路，這是解決 `NoRouteToHostException` 的關鍵。
*   **LM Studio 伺服器狀態**：確保遠端 LM Studio 已啟動，Port 設定為 `1234`，並已載入模型且開啟了 `Local Server`。
*   **網路連通性**：確認本地 Mac mini 與遠端 PC 之間的網路暢通，無防火牆阻擋。
