# 2026-04-06 專案測試與環境優化日誌

## 工作摘要
今日主要針對 `csp-portal-web` 專案進行了測試覆蓋率的提升、測試引擎的升級以及 Maven 環境的相容性修復。成功將有效覆蓋率從 0% 提升，並建立了一套穩定的單元測試基礎。

## 執行的關鍵任務

### 1. 測試覆蓋率優化
- **排除 DTO 與 Entity**：在 IntelliJ IDEA 的 Run Configuration 中設定了 Exclude Pattern，成功排除了無商業邏輯的 Data Objects，使覆蓋率數據更具參考價值。
- **解決語法衝突**：修正了 `**` 萬用字元在某些 IntelliJ 版本中造成的 `PatternSyntaxException` 正則表達式報錯，改用相容性更好的單星號或正則語法。

### 2. 測試引擎升級 (JUnit 4 -> JUnit 5)
- **問題診斷**：發現 `pom.xml` 中排除了 `junit-vintage-engine`，導致 IntelliJ 的預設引擎無法辨識舊版的 JUnit 4 測試案例，引發 "No test found" 錯誤。
- **全面升級**：將所有測試案例遷移至 JUnit 5 (Jupiter)，包含 `@Test`、`@BeforeEach` 以及 `Assertions` 的全面更新。

### 3. 核心邏輯單元測試 (共 12 個案例)
- **EmployeeControllerTest** (4 案)：驗證部門清單讀取、搜尋委派與頁面導向邏輯。
- **SqlControllerTest** (1 案)：利用反射技術測試私有方法 `processForMyBatis`，確保 SQL 分號自動過濾邏輯正確，有效防止 MyBatis 執行錯誤。
- **ThymeleafLayoutInterceptorTest** (3 案)：測試頁面佈局切換機制，涵蓋 `@Layout` 註解處理與 Redirect 排除邏輯。
- **JXLExcelBuilderTest** (2 案)：驗證 Excel 報表生成邏輯，並解決了專案中同時存在 `javax.servlet` 與 `jakarta.servlet` 的套件相容性挑戰。
- **QuestionControllerTest** (2 案)：模擬 ChatGPT 問答流程與 SSE (Server-Sent Events) 串流轉發。

### 4. Maven 環境配置修正
- **Surefire Plugin 更新**：將 `pom.xml` 中的 `maven-surefire-plugin` 版本提升至 `2.22.2`，確保 Maven CLI 模式下也能正確識別並執行 JUnit 5 測試。

## 目前狀態
- **測試結果**：12 個測試案例全部綠燈通過 (`BUILD SUCCESS`)。
- **開發工具**：IntelliJ 現在可以順利執行 "Run with Coverage"，並產出精確的邏輯覆蓋率報告。
